Getting Cat (catlang)

Overview of the Cat language

An overview of the cat language

The Cat programming language, developed by Christopher Diggins and kindly placed in the public domain by him, is a member of the class of stack languages that are derived ultimately from Charles H. Moore's seminal stack-oriented language FORTH. The "Cat" language is described in detail by Diggins on his website. The purpose of this documentation is to describe extensions and alterations to the language that result from a re-implementation of the language in Python 2.x by Andrew Nelis. (Note: the Cat image above is from Christopher Diggins's website.)

Getting and installing catlang

The catlang package is available on github. If you have git installed on your system, you can clone the repository by using the command line: git clone https://github.com/AndrewNelis/catlang.git. If, however, you don't have git installed you can down load the ".zip" file and unpack it in the usual way.

File Layout

The top-level layout of files and directories is shown above. The contents of each of the sub-directories will be described below, but for now the four files will be described. The LICENSE.txt file is the usual boilerplate for the LGPL. The README.md is a MarkDown file describing authorship of the current version of the catlang "system." The real work begins with the "catlang.cfg" configuration file. This is the configuration file for the catlang "system." It follows the usual ".ini" file format and is parsed by the Python 2.x "ConfigParser" module in the NS.py file.

The catlang.py file

The picture above shows the start of the catlang.py file. This file is the interactive driver for catlang. It indicates that there are two ways to run it:

./catlang.py --eval "code"
./catlang.py

The first evaluates the specified code and returns. For example:

catlang.py --eval "1 2 +"
===> 3

To run in interactive mode:

./catlang.py

catlang.cfg

Shown above is the bulk of the configuration file. Not shown are the output colour choices (which are obvions) and control of the startup message. A few note may be in order:

paths:catdefs -- path to where additional Cat definitions (built-in and user-defined) may be found
paths:browser -- where the documentation you are now reading is found
paths:load_file -- a path to a Cat file that is to be loaded after the basic initialization of the cat language
link:namespaces -- a comma-separated list of namespasces that are to be linked to the 'user' namespace after initialization (used in conjunction with load_file, above)
prompt:default -- the default (startup) terminal prompt
errors:full_error_info -- sets the depth of information reported when an error is encountered by Cat. The choices are:

  • off -- announces only the error
  • on -- announces error and shows a condensed backtrace
  • super -- announces error, shows a full backtrace, and quits cat.

stack:show_stack -- controls whether or not the stack is to be displayed after executing an interactive Cat line
stack:use_alt_format -- if stack is to be shown, the usual output format is to print the elements inline with the top of the stack to the far right. However, this option permits printing the stack contents one per line with the top of the stack first.
display:user_colour -- use coloured output on the terminal if the value is 'true', or uncoloured if the value is 'false'. The default is to use colour. The following options define where colour is to be used:

  • display:prompt -- the colour associated with the prompt and input line (default is empty which means black)
  • display:info -- the colour associated with informational output (such as help) (default is 'green')
  • display:error -- the colour associated with error messages (default is 'red')
  • display:stack -- the colour associated with stack output (default is 'blue')

Available colours are

  • grey
  • red
  • green
  • yellow
  • blue
  • magenta
  • cyan
  • white

Finally the startup messasge can be controlled:
misc:show_MOTD -- show the startup message (found in cat/repl.py) The default is 'false'

Contents of the "cat" directory

These files constitute the catlang engine. Here, in a general sort of way, is what they do:

NS.py -- handles all the high-level namespace activity and is the place where the catlang engine is initialized and various definitional files are linked in to the engine.
eval.py -- evaluates catlang "programs" input from the terminal
namespace.py -- low-level functions for manipulating namespaces
parser.py -- parses input text for use by the catlang eval functikon
repl.py -- the read-evaluate-print-loop code
stack.py -- handles all stack-related functions
__init__.py -- a package marker file for Python's use (currently is empty)

Contents of the defs directory

Zoom

This directory contains all of the Python-language files needed to flesh out the catlang engine. In general terms here is what each does:

__init__.py -- defines the directory as a Python module and imports all of the other .py files into Python's "sys" module for general access
cat_arithmetic.py -- defines all of the arithmetic operations available in catlang (with many extensions from Python)
cat_basic_tests.py -- runs tests on over 150 catlang "words" (functions)
cat_conditionals.py -- defines all of the Cat conditionals and a few extra ones based on Python
cat_control.py -- defines all control "words" (functions), such as "if", "repeat", "while", etc.
cat_debug.py -- defines a few helpful debugging tools
cat_io.py -- defines "words" (functions) to handle file and terminal input/output
cat_lists.py -- defines "words" (functions) that deal with lists
cat_meta.py -- defines "words" (functions) that in a sense manipulate the catlang system. Examples are "load", "doc", "help", etc.
cat_misc.py -- defines "words" (functions) that don't seem to fit well into other categories
cat_nsWords.py -- defines "words" (functions) that deal with namespaces
cat_stack.py -- defines "words" (functions) that deal with the catlang stack
cat_strings.py -- defines "words" (functions) that manipulate strings
cat_tagExpr.py -- a support file that defines a simple recursive descent class for parsing logical expressions involving catlang tags
cat_time.py -- defines "words" (functions) that work with time
cat_types.py -- defines "words" (functions) to work with the various object types in catlang
cat_user.py -- a place to put user-defined Python language catlang functions

All of these files are loaded automatically into the catlang "system" when the engine is initialised.

Content of the CatDefs directory

Zoom

In this directory more catlang "words" (functions) are found that can be optionally loaded into the "system." The file names are pretty self-explanatory, with perhaps those starting with 'demo_' and the one called 'everything.cat'. The files starting with demo_' are exactly that: a couple of demonstrations of fairly sophisticated uses of catlang. The 'everything.cat' file is a convenience for loading all of these auxilliary files (exception the "demo_" ones) into the "system" at one time.

Contents of the docs directory

This directory contains the HTML for the browser-based general help system.

Contents of the termcolor directory

This directory contains the code for Konstantin Lepa's terminal coloration package. It is included for convenience. It will need to be inserted into the main site package directory by using the usual incantation:

catlang$ cd termcolor
termcolor$ sudo python setup.py install
